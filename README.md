# Steam App ID Extractor

A simple Python utility that scans `.lua` files downloaded from Openlua.cloud inside a folder and extracts **Steam App IDs** from specific patterns, then generates a formatted report automatically.

## Features

- 🔍 Recursively scans folders and subfolders
- 📄 Reads all `.lua` files
- 🎮 Extracts Steam App IDs from:
  - `addappid(<ID>)`
  - `setManifestid(<ID>)`
  - `-- AppID <ID>`
- 📝 Captures inline comments as optional notes
- 🧹 Filters generic autogenerated comments
- 📊 Sorts extracted IDs numerically
- 💾 Saves results automatically to:

```text
~/extracted_steam_ids.txt
```

## Requirements

- Python 3.6+

No external dependencies required.

## Installation

Clone or download the script:

```bash
git clone <repository-url>
cd <repository-folder>
```

Or simply download:

```text
id_extractor.py
```

## Usage

Run the script:

```bash
python id_extractor.py
```

The program will ask:

```text
Enter the path to the folder containing the .lua files:
```

Example:

```text
Enter the path to the folder containing the .lua files:
/home/user/SteamConfigs/
```

The script will scan all `.lua` files recursively.

## Example Input

Example `.lua` file:

```lua
addappid(570) -- Dota 2
addappid(730) -- Counter-Strike 2
-- AppID 440 Team Fortress 2
setManifestid(620)
```

## Example Output File

Generated report:

```text
=== EXTRACTED STEAM APP IDS REPORT ===

File: games.lua
--------------------------------------------------
ID: 440 | Note: Team Fortress 2
ID: 570 | Note: Dota 2
ID: 620
ID: 730 | Note: Counter-Strike 2
```

Saved to:

```text
~/extracted_steam_ids.txt
```

## Error Handling

The script automatically handles:

- Invalid directories
- File read errors
- Encoding issues (`utf-8` with ignored errors)
- Empty scan results
